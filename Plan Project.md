«Автоматизированная инвентаризация полезащитных лесополос».

**Этап 1. Постановка задачи и аналитика требований**

1.  Уточнить определение «лесополосы»: минимальная/максимальная ширина (2–60 м), ориентация, пространственная структура, породный состав.

2.  Формализовать критерии качества:

- структурные: сомкнутость крон (в %), ширина, непрерывность (линейность без разрывов), наличие сухостоя, захламлённость, высота;

- санитарное/функциональное состояние: NDVI-статус (высокий/угнетённый), распад, усыхание, незаконная вырубка.

- интегральная оценка: категории «хорошая», «удовлетворительная», «плохая» на основе агрегированных правил;

- требования к координатам: точность не хуже планового положения пикселя снимка, системы координат WGS84/UTM. Режим вывода – векторный полигон или линия с атрибутами.

- масштаб проекта: площадь мониторинга, периодичность обновления снимков, необходимость исторической ретроспективы;

1\. Выбор источника спутниковых данных:

2\. Сбор данных за вегетационный период (пик зелёной массы) и безоблачные сцены. Загрузка через API Copernicus, Google Earth Engine.

3\. Формирование калибровочных/тестовых полигонов с высокой изменчивостью ландшафта (лесостепь, степь, разная ориентация лесополос, разные породы).

**Этап 3. Разметка и создание эталонного датасета**

1\. Разметка экстента лесополос:

- инструмент: Labelme или CVAT с геопривязкой.

- тип разметки: полигональная маска (сегментация) или полилинии с буфером;

- сегментация с тонкими границами (опционально distance transform для повышения устойчивости).

2\. Разметка качества:

- создание трёх классов состояния на основе фотоэталонов: «хорошее», «удовлетворительное», «плохое».

- альтернативно — числовые атрибуты (проективное покрытие, ширина, NDVI) для регрессионной задачи.

3\. Формирование выборок:

- разбивка на train / val / test (70/15/15) с пространственной стратификацией.

- балансировка по типам рельефа, сезонам, углам солнечного освещения.

- препроцессинг: выравнивание гистограмм, атмосферная коррекция, тайлинг с перекрытием.

**Этап 4. Разработка и обучение нейросетевой модели распознавания лесополос**

1.  Выбор архитектура базовой сегментационной модели:

> Первый кандидат: U-Net++ с эффективным энкодером (ResNet-50, EfficientNet-B3) — хорош для линейных структур.

Второй кандидат: DeepLabV3+ с atrous convolution — улавливает мультимасштабный контекст.

Третий кандидат: трансформерный сегментатор (SegFormer, Swin-UNet) — для тонкого учёта текстуры.

2\. Учёт спектральных особенностей: использование не только RGB, но и NIR, Red Edge, SWIR. Сборка мультиспектральных тензоров.

3\. Лосс-функции: комбинация DiceLoss + Focal Tversky Loss (чтобы не игнорировать узкие объекты), плюс Boundary Loss для улучшения контуров.

4\. Аугментации: геометрические (повороты на произвольный угол, отражения, эластичная деформация), цветовые (яркостные, спектральный сдвиг), а также CutMix/Mosaic на основе лесополос.

5\. Стратегия обучения: предварительное обучение на больших наборах <span class="mark">(BigEarthNet/SSL4EO)</span>, тонкая настройка под задачу, мониторинг метрик (IoU, F1 на классе «лесополоса», Precision-Recall).

6\. Борьба с дисбалансом: взвешенный сэмплинг плиток с наличием лесополос, использование потерь с фокусом.

**Этап 5. Постобработка маски и векторизация**

1\. Фильтрация предсказаний: удаление шумовых артефактов (морфологическое закрытие/открытие, минимальная площадь связных компонент).

2\. Скелетизация и полигонизация:

- построение скелета (medial axis) для получения линейной оси лесополосы;

- реконструкция буфера вдоль скелета с учётом локальной ширины предсказанной маски — получение чистого полигона;

- выделение отдельных лесополос: разрезка по пересечениям, объединение мелких звеньев в одно целое по линейному продолжению.

**Этап 6. Определение качества лесополос**

1\. Первичные количественные метрики на основе маски и мультиспектральных данных:

- средняя ширина полосы (метры) по поперечному профилю масок;

- коэффициент сомкнутости = отношение площади маски к площади ограничивающего полигона вдоль скелета;

- индекс непрерывности = длина главной оси / общая длина обнаруженных фрагментов;

- средний NDVI (и/или LAI) внутри маски, гистограмма значений;

- доля пикселей «сухостоя» (низкий NDVI, высокий SWIR), доля прогалин.

2.  Классификатор состояния:

Вариант А: обучение отдельной нейросети-классификатора (ResNet-18/ViT) на вырезанных фрагментах лесополос (по предсказанным маскам) для отнесения к классам качества.

Вариант Б: решающие правила на основе комбинации метрик (например, дерево решений) — прозрачный и легко валидируемый подход для пилотного запуска.

3.  Агрегация в итоговую качественную оценку) с возможностью расшифровки факторов.

**Этап 7. Геопространственная привязка и экспорт координат**

1\. Преобразование пиксельных координат в мировые: чтение геотрансформа (GDAL) исходного.

2\. Формирование геометрий:

- ось лесополосы (LineString), полигон лесополосы (Polygon) с сохранением UTM/WGS84;

- присваивание каждому объекту уникальный ID и атрибуты: ширина, сомкнутость, NDVI, качество, дата снимка, источник.

- валидация геометрии: устранение самопересечений, контроль топологии (пересечения с дорогами и полями не должны разрушать целостность).
