# Требования первого этапа

Документ фиксирует результат этапа "Постановка задачи и аналитика требований" для проекта автоматизированной инвентаризации полезащитных лесополос.

## Цель проекта

Построить пайплайн, который по спутниковым снимкам обнаруживает полезащитные лесополосы, оценивает их состояние и формирует геопространственный слой для мониторинга, планирования обследований и принятия решений о восстановлении насаждений.

## Рабочее определение лесополосы

Полезащитная лесополоса в рамках проекта - это линейное древесно-кустарниковое насаждение искусственного или полуискусственного происхождения, расположенное среди сельскохозяйственных угодий, вдоль границ полей, дорог, водотоков или иных протяженных объектов и выполняющее защитную функцию.

Минимальные признаки объекта:

- ширина от 2 до 60 м;
- выраженная линейная форма с длиной, существенно превышающей ширину;
- наличие древесной или древесно-кустарниковой растительности;
- пространственная непрерывность либо цепочка фрагментов, которую можно интерпретировать как одну полосу;
- геометрия, пригодная для вывода как Polygon и/или LineString.

Объекты, которые не считаются целевыми без дополнительного подтверждения:

- компактные лесные массивы без линейной структуры;
- одиночные деревья и малые группы деревьев вне протяженной полосы;
- сады, парки и внутрипоселковые зеленые насаждения;
- естественные леса, если они не могут быть отделены как защитная линейная посадка.

## Геометрические требования

| Параметр | Базовое требование |
| --- | --- |
| Минимальная ширина | 2 м |
| Максимальная ширина | 60 м |
| Ориентация | любая; модель не должна быть привязана к сторонам света |
| Пространственная структура | одно- или многорядная посадка, непрерывная полоса, фрагментированная полоса с разрывами |
| Породный состав | фиксируется как дополнительный атрибут при наличии данных; не является обязательным входом для сегментации |
| Основная геометрия | Polygon |
| Дополнительная геометрия | LineString по оси лесополосы |

## Показатели качества

Структурные показатели:

- средняя ширина лесополосы, м;
- сомкнутость крон, %;
- непрерывность линейной структуры;
- наличие разрывов и прогалин;
- наличие сухостоя;
- захламленность при наличии полевых или высокодетальных данных;
- высота насаждений при наличии LiDAR, стереосъемки или внешних источников.

Санитарные и функциональные показатели:

- NDVI-статус: высокий, средний, угнетенный;
- признаки распада и усыхания;
- доля пикселей с низким NDVI и высоким SWIR как индикатор сухой растительности;
- признаки незаконной вырубки или резкого сокращения площади по историческим снимкам;
- стабильность защитной функции через непрерывность и сомкнутость.

## Интегральная оценка состояния

Для пилотного запуска используется прозрачная схема на основе правил. Пороговые значения являются стартовыми и должны уточняться на эталонных полигонах и полевых данных.

| Категория | Условия |
| --- | --- |
| Хорошая | сомкнутость крон >= 70%, NDVI >= 0.55, непрерывность >= 0.85, прогалины <= 10%, сухостой <= 10% |
| Удовлетворительная | сомкнутость крон 40-70%, NDVI 0.35-0.55, непрерывность 0.60-0.85, прогалины 10-30%, сухостой <= 25% |
| Плохая | сомкнутость крон < 40%, NDVI < 0.35, непрерывность < 0.60, прогалины > 30% или сухостой > 25% |

Если показатели попадают в разные категории, итоговая оценка выбирается по наиболее ограничивающему фактору. В результате нужно сохранять не только класс, но и список факторов, которые повлияли на оценку.

## Требования к координатам и форматам

- точность координат не хуже планового положения пикселя исходного снимка;
- поддерживаемые системы координат: WGS84 для обмена и UTM для метрических расчетов;
- все метрические показатели рассчитываются в проекционной системе координат;
- экспортируемые геометрии должны проходить проверку валидности и не содержать самопересечений;
- основной формат обмена: GeoJSON или GeoPackage;
- для каждого объекта должен формироваться уникальный идентификатор.

Минимальный набор атрибутов результата:

- `belt_id` - уникальный идентификатор лесополосы;
- `geometry_type` - polygon или centerline;
- `width_m` - средняя ширина;
- `crown_closure_pct` - сомкнутость крон;
- `continuity_idx` - индекс непрерывности;
- `mean_ndvi` - среднее значение NDVI;
- `dry_share_pct` - доля сухостоя или сухой растительности;
- `gap_share_pct` - доля прогалин;
- `quality_class` - хорошая, удовлетворительная или плохая;
- `quality_factors` - факторы, повлиявшие на оценку;
- `image_date` - дата снимка;
- `source` - источник данных;
- `crs` - система координат результата.

## Масштаб мониторинга

Перед запуском обработки для новой территории нужно зафиксировать:

- границу территории мониторинга;
- площадь территории;
- источник и разрешение снимков;
- период съемки, предпочтительно пик зеленой массы;
- допустимый уровень облачности;
- периодичность обновления;
- необходимость исторической ретроспективы для выявления вырубок и деградации.

Базовая конфигурация пилота:

- территория: один тестовый район или набор сельскохозяйственных контуров;
- сезон: вегетационный период;
- обновление: один раз в год либо чаще при наличии новых безоблачных сцен;
- ретроспектива: минимум два периода при задаче выявления вырубок и деградации.

## Открытые вопросы

- точная территория пилотного мониторинга;
- приоритетный источник снимков и пространственное разрешение;
- наличие полевых обследований для калибровки качества;
- необходимость отдельного учета породного состава;
- правила обработки пересечений с дорогами, каналами и границами полей.
